# Ticket Lifecycle

## Lifecycle Used in This Lab

- New
- In Progress
- Resolved
- Escalated

## Status Meanings

### New

The ticket has been created and is waiting for first-pass reasoning or triage.

### In Progress

The case is being actively worked through. This can include first-pass reasoning, critique, ticket write-up, and concept clarification.

### Resolved

A supported simulated outcome has been selected, documented, and verified within the lab's scope.

### Escalated

The case has reached a justified handoff point because the issue is beyond the current support boundary or because the available evidence is not enough to resolve it at the current level.

## Status Discipline

- move to `In Progress` once the case is actively being reasoned through
- use `Resolved` only when a chosen resolution is documented clearly enough to explain what fixed the issue and how success was verified
- use `Escalated` only when the handoff is justified by scope, evidence, or support boundary
